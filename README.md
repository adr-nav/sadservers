# SadServers Solutions
Repository with my solutions to the practice scenarios from [SadServers](https://sadservers.com/scenarios), a platform for training Linux and DevOps troubleshooting skills.

## What is SadServers?
SadServers provides preconfigured Linux servers with a specific problem that needs to be diagnosed and resolved within a time limit, simulating real-world sysadmin/SRE situations.

## Repository structure
sadservers/

├── 01-easy/

│   ├── scenario-name/

│   │   ├── README.md       # problem description and solution

│   │   └── solution.sh      # commands/script for the solution

├── 02-medium/

│   └── ...

├── 03-hard/

│   └── ...


## Solution format
Each scenario includes:
- **Problem statement**: summary of the proposed problem.
- **Diagnosis**: steps taken to identify the cause.
- **Solution**: commands or script applied to solve it.
- **Notes**: lessons learned, alternatives, or useful references.

## Disclaimer
These solutions are illustrative and reflect my personal approach; other equally valid ways of solving each scenario may exist. The goal is to document the learning process, not to serve as "the only answer".
